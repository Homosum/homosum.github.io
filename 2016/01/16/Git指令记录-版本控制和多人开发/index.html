Git指令记录(版本控制和多人开发)

Git在版本控制和多人开发中的使用，整理了用的比较多的指令如下：

git commit -am 'xxxx'   //提交
git push origin master //推送
git remote -v //查看远程库地址
git init
git remote add origin source URL
git pull origin master
git push origin master
git statu -a//
git branch
git branch tryidea//创建branch
git branch//查看branch
git check out tryidea//切换branch
git check out master
git branch -d tryidea//删除branch
git branch -b tryidea//创建并切换到分支
git commit -am ‘commit message’//提交
git merge tryidea//把分支合并到master上
git branch -d tryidea
git push origin develop:master -f develop分支强推到主分支，以前的主分支指针指向develop
git add
git reset

git rm cached 文件路径
git clean -d -fx
//移除不跟踪文件

git fetch origin version2:version2 从远程获取version2分支，并在本地创建的version2分支保存下来

在当前分支 git merge origin version2 将version2分支合并到当前分支，形成更完整的当前分支
